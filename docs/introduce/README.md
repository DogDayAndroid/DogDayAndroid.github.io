# DogDayAndroid项目介绍

## 绪论

### 1.1 研究的目的和意义
目前 Android 已经成为最流行的移动操作系统,巨大的市场占有率带来了极大的利润,大量的攻击者也将目光投到了 Android 平台.阿里移动安全2016年第二季度的报告显示．该季度感染恶意代码的 Android 设备总量高达2877万,比上一季度增长96．2％.与此同时,原本用来防止应用程序被逆向分析、破解修改、重打包的加壳技术也被恶意程序利用攻击者利用加壳技术隐藏木马病毒程序的真正代码来躲避安全软件杀毒引擎的扫描与移动安全研究人员的分析.

目前Android 平台常用的加壳技术主要包括:

- 虚拟机检测
- 代码动态加载
- 代码动态修改
- 反调试

无论是安全人员分析应用还是各种杀毒引擎扫描查杀恶意程序.首先都必须获得原始的[可执行文件(Dex)](http://www.android-doc.com/reference/dalvik/system/DexFile.html).

由于脱壳技术对于 Android 应用安全的重要作用,百度移动安全的研究人员提出并实现了基于 Xposed 框架的脱壳工具 [ZjDroid](https://xueshu.baidu.com/usercenter/paper/show?paperid=5154c35ae2e7dd9f8763f725f2ac75bd&site=xueshu_se),安全研究员瘦蛟舞基于 CydiaSubstrate 框架实现了 [DexMemoryDumpTools](https://github.com/WooyunDota/DumpDex).其中的基本思想就是 Android 应用运行的内存空间中存在完整的 Dex 文件,直接将对应区域的内存读出即可.但是由于加壳技术的发展,当加壳过后的程序检测到在上述脱壳环境中运行时,可以通过直接退出、大量形成虚假的 Dex 内存空间、Dex 动态修改等对抗手段让其失效．使得我们要么不能得到 Dex,要么得到错误或不完整的 Dex.传统内存 dump 出 Dex 的脱壳软件对于加固厂商的新型壳已经不适用。

我们的目标是通过基于 OS 或驱动层面,研发一种多层次的脱壳系统,从更深的系统维度去抵抗新型壳。我们首先通过研究整体Dump获取Dex文件的技术，并结合java的反射机制获取安卓程序运行过程中的类以及函数，通过主动构造参数来生成类或调用函数，获取类及函数原本的CodeItem。通过修补CodeItem和回填到Dex文件中获取完整的Dex文件，来达到脱壳的目的。

### 1.3 主要研究内容

我们的主要研究内容是寻找一个恰当的脱壳时机进行脱壳，并将函数类和函数体进行抽取和回填。我们通过在安卓应用启动时附加一个子进程来监控程序运行过程，并通过它来对当前程序内容进行反射提取，从而完成脱壳和类以及函数内容体的抽取回填。

我们通过修改和重构当前流行的的GSI系统PixelExperience中的ART运行时环境来达成脱壳效果，并提供了一个用于用户进行系统刷机的安卓镜像。

我们的贡献可以总结如下：

1. 我们提供了一个新的基于主动调用的脱壳方法。

2. 我们使用GSI来编译系统镜像使得脱壳镜像使得我们的系统镜像无关于设备可以通用在更多安卓设备上。

3. 在现有的脱壳实践中，我们脱壳出来的文件更加贴近原本的安卓软件源Dex内容。

4. 我们的实验结果显示，对脱壳的软件的还原准确率达到了xx%。

本文的系统的意义是能够对程序提供进行深入和细粒度的脱壳还原。
